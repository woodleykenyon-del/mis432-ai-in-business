Course Description A/B Test

A simulation-based experiment comparing two course description formats during university registration. Which format leads more students to express enrollment intent?

Scenario

What we are testing

A university is evaluating two formats for course descriptions shown during online registration. Version A presents information as a traditional paragraph, which is the current default. Version B uses a bullet-point format that highlights key learning outcomes at the top. The goal is to determine which format leads more students to click "Add to Plan," which the university uses as a proxy for enrollment intent.

Version A — Control

Traditional Paragraph Format

This course introduces students to the principles of relational database design, SQL querying, and data management for business applications. Students will work with industry-standard tools and develop skills in organizing, retrieving, and analyzing structured data. Recommended for students interested in information systems or data-intensive careers.
Assumed conversion rate: 12%
Version B — Treatment

Bullet-Point Outcomes Format

MIS 320: Database Systems
You will learn to:
  • Design relational databases using ER diagrams
  • Write SQL queries to extract and analyze data
  • Apply database skills to real business problems
Recommended for: MIS, Business Analytics, Data-focused careers.
Assumed conversion rate: 17%

The 5-percentage-point difference in assumed conversion rates reflects the hypothesis that students respond better to specific, scannable outcomes than to dense paragraphs. The simulation tests whether that difference, if observed in data, would be statistically meaningful.

Simulation Setup

Parameters and methodology

This simulation uses a seeded random number generator so the results are stable and reproducible on every page load. Two groups of 500 students each are independently simulated. Each student in Group A is assigned a click outcome based on a 12% probability; each student in Group B is assigned a click outcome based on a 17% probability.

The statistical test is a two-proportion z-test, which compares the two observed conversion rates and calculates the probability of seeing a difference this large if there were truly no difference between the formats. A p-value below 0.05 is used as the threshold for statistical significance.

Statistical Interpretation

What the numbers mean

The p-value tells you how likely it would be to observe a gap this large between the two groups purely by chance, assuming no real difference exists. A p-value below 0.05 means that kind of chance gap would happen fewer than 5% of the time, which is the conventional threshold for calling a result statistically significant.

If the simulation returns a p-value below 0.05, it means the bullet-point format appears to meaningfully outperform the paragraph format for driving enrollment intent. The difference is unlikely to be noise.

If the p-value is close to but not below 0.05, the result is borderline. The simulation would not strongly support a rollout decision, and collecting more data would be the better path.

About the z-statistic

The z-statistic measures how many standard errors apart the two conversion rates are. Larger absolute values indicate a more meaningful difference. A z-statistic above 1.96 corresponds to a two-tailed p-value below 0.05.

The Prediction-Decision Gap

Even a statistically significant result does not automatically tell the university to change every course description. Statistical significance only answers one question: is this difference likely real? It does not answer whether the difference is large enough to justify a catalog-wide rollout, whether the effect persists across all subject areas and student populations, or whether a higher "Add to Plan" click rate actually leads to higher enrollment, better persistence, or improved student outcomes. The university would still need human judgment to weigh the cost of re-formatting thousands of descriptions against the potential benefit, and to consider whether the format alone drives the difference or whether something else (like the example course or the specific wording) accounts for it. A significant prediction is a reason to take the question seriously, not a reason to stop asking it.

Reflection

Process and limitations

I built a simulated A/B test for a university course registration decision. The test compares a traditional paragraph course description against a bullet-point format with key outcomes highlighted. I used a simulated sample of 1,000 students, split evenly between control and treatment, then measured which format led to higher enrollment intent. After the first version was generated, I adjusted the scenario to make it more specific to a university context and made the output explain the business decision instead of only showing numbers.

This experiment is useful, but it does not capture everything a real university would need to know. A real test would need to account for course popularity, instructor reputation, time conflicts, major requirements, and whether students actually enroll later instead of just clicking interest in the moment. Even if the result is statistically significant, the prediction-decision gap still applies because a better click rate is not automatically the same thing as a better academic outcome. The university would still need human judgment before changing every course description format across the catalog.
