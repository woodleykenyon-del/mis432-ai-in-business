Business Problem

Why the image Netflix shows you matters

Netflix has a fundamental engagement problem: users scroll through an enormous catalog, and the first thing that influences whether they click on a title is often its artwork. Which image gets shown matters enormously. Netflix recognized early that no amount of human intuition can reliably predict which visual will resonate with millions of diverse viewers — and that the cost of getting it wrong is measured in clicks, viewing time, and platform engagement.

Different images surface different emotions, characters, and themes. One poster might make a drama feel gripping and urgent; another might make it feel slow and forgettable. Because people react to visuals differently, the only way to know what actually works is to test it with real users.

This is why Netflix shifted artwork selection from subjective human judgment to a data-driven experimentation system. That shift is an example of automating a decision rather than merely augmenting one. When a person reviews test results and picks the winner, the system is augmenting their judgment. When the platform automatically selects and serves the higher-performing image to millions of users based on statistical evidence, the system is making the decision. That distinction matters at scale: automated decisions move faster and reach further, which raises the stakes for accuracy, fairness, and ongoing monitoring.

The shift probably made artwork selection more measurable and more responsive to actual user behavior. But it also raised legitimate questions — about whether short-term click metrics can crowd out longer-term creative or brand considerations, and whether the system's outputs are being interpreted and monitored carefully enough.

Methodology

Why A/B testing — and why it works

A/B testing works because of one foundational principle: random assignment. Users are placed into groups — Artwork A or Artwork B — by chance, not by profile or prior behavior. When the two groups are comparable at the start, any difference in outcomes can be attributed to the thing being tested: in this case, the artwork.

Without random assignment, the results can be systematically misleading. If more engaged or more curious users ended up in one group by accident, that group might produce a higher click-through rate regardless of which artwork it saw. The experiment would confuse user characteristics with artwork quality, and Netflix could make the wrong rollout decision based on a flawed comparison.

Sample size matters for similar reasons. A small experiment can swing heavily due to random variation, making one version look better when there is no real difference. Larger samples produce more stable estimates — the kind needed to justify platform-wide decisions.

To evaluate the experiment, three things were needed: how many users saw each version, how many clicked in each group, and whether the observed difference in click-through rate was statistically meaningful or likely due to chance.

What I Built

A simulated A/B experiment in Python

I built a simplified experimentation system in Python that mirrors the core logic of Netflix's artwork testing process. One thousand users were randomly assigned to either Artwork A or Artwork B using NumPy. Each user either clicked or did not, based on defined click probabilities — 24% for Artwork A and 30% for Artwork B. The results were stored in a structured pandas DataFrame and analyzed using a two-proportion z-test from the statsmodels library.

The simulation captures the essential structure of a real experiment: random exposure, outcome measurement, statistical comparison, and a clear business question at the end. It is a small-scale classroom version of a decision system that Netflix runs on millions of users simultaneously — which creates its own challenges around data volume, experiment interference, and result interpretation across user segments.

Netflix's real system adds considerably more complexity: personalized artwork selection based on viewing history, concurrent experiments running on overlapping user populations, and downstream tracking of whether higher clicks actually lead to longer viewing or improved retention. This simulation captures the decision logic while leaving those layers for future work.

Statistical Interpretation

What the numbers actually mean

The p-value from the z-test was 0.0127. In plain English: if there were truly no difference between the two artwork versions, you would expect to see a gap this large by random chance only about 1.3% of the time. Because 0.0127 falls below the conventional threshold of 0.05, the result is considered statistically significant.

That is meaningful, but it does not mean the experiment is fully conclusive on its own. A statistically significant result tells you the difference is unlikely to be noise — it does not tell you that Artwork B will perform better in every context, with every user segment, or over a longer time horizon.

Before recommending a full platform rollout, it would be worth checking whether the result holds across different user segments, whether higher clicks translate into longer viewing sessions or improved retention, and whether the result is consistent across repeated experiments. Acting too quickly on a single statistically significant result can lead to rolling out a version that looked good in one test but does not actually improve broader business performance over time.

The bar chart produced in the analysis makes the difference immediately visible — which matters in a business setting. When data is communicated clearly and visually, it helps non-technical stakeholders understand the takeaway without needing to interpret raw numbers. Clear visuals help turn analysis into action.

Source Code

Full Python Code

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from statsmodels.stats.proportion import proportions_ztest

# Set random seed for reproducibility
np.random.seed(42)

# Step 1: Simulate the experiment
n_users = 1000
user_ids = np.arange(1, n_users + 1)

# Randomly assign users to Artwork A or Artwork B
artwork_versions = np.random.choice(['A', 'B'], size=n_users)

# Define click probabilities
click_probs = {'A': 0.24, 'B': 0.30}

# Simulate clicks based on assigned artwork version
clicked = [np.random.binomial(1, click_probs[version]) for version in artwork_versions]

# Create DataFrame
df = pd.DataFrame({
    'user_id': user_ids,
    'artwork_version': artwork_versions,
    'clicked': clicked
})

# Step 2: Measure performance
summary = df.groupby('artwork_version').agg(
    total_users=('user_id', 'count'),
    total_clicks=('clicked', 'sum')
)

summary['ctr'] = summary['total_clicks'] / summary['total_users']
summary['ctr_percent'] = (summary['ctr'] * 100).round(2)

print("Experiment Summary:")
print(summary[['total_users', 'total_clicks', 'ctr_percent']])
print()

# Step 3: Visualize click-through rate
plt.figure(figsize=(8, 5))
bars = plt.bar(summary.index, summary['ctr'])

plt.title('Click-Through Rate by Artwork Version')
plt.xlabel('Artwork Version')
plt.ylabel('Click-Through Rate')
plt.ylim(0, max(summary['ctr']) + 0.05)

for bar, value in zip(bars, summary['ctr']):
    plt.text(
        bar.get_x() + bar.get_width() / 2,
        value + 0.005,
        f"{value:.2%}",
        ha='center'
    )

plt.show()

# Step 4: Statistical test (two-proportion z-test)
click_counts = summary['total_clicks'].values
group_sizes = summary['total_users'].values

z_stat, p_value = proportions_ztest(count=click_counts, nobs=group_sizes)

# Step 5: Business interpretation
ctr_a = summary.loc['A', 'ctr']
ctr_b = summary.loc['B', 'ctr']
ctr_diff = ctr_b - ctr_a

print(f"Z-statistic: {z_stat:.4f}")
print(f"P-value: {p_value:.4f}")
print()
print(f"CTR for Artwork A: {ctr_a:.2%}")
print(f"CTR for Artwork B: {ctr_b:.2%}")
print(f"Difference in CTR (B - A): {ctr_diff:.2%}")
print()

alpha = 0.05
if p_value < alpha:
    print("Conclusion: The difference in click-through rates is statistically significant.")
    if ctr_b > ctr_a:
        print("Business Recommendation: Artwork B performed better and would be the stronger candidate for rollout.")
    else:
        print("Business Recommendation: Artwork A performed better and would be the stronger candidate for rollout.")
else:
    print("Conclusion: The difference in click-through rates is not statistically significant.")
    print("Business Recommendation: Do not make a full rollout decision yet. Collect more data before switching.")
